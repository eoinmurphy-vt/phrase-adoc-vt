NeuVector s'engage à informer la communauté des problèmes de sécurité. Vous trouverez ci-dessous une liste de référence des avis de sécurité publiés et des CVE (Common Vulnerabilities and Exposures) pour les problèmes que nous avons résolus.

== Liste CVE

|===
| ID | Description | Date | Libération

| https://github.com/neuvector/neuvector/security/advisories/GHSA-8pxw-9c75-6w56[CVE-2025-8077] 
| Pour le déploiement de NeuVector sur l'environnement basé sur Kubernetes, le mot de passe bootstrap de l'utilisateur admin par défaut sera généré aléatoirement et stocké dans un secret Kubernetes. L'administrateur par défaut devra d'abord obtenir le mot de passe de démarrage à partir du secret Kubernetes et sera invité à changer de mot de passe après la réussite de la première connexion à l'interface utilisateur.
| 25 août 2025

| https://github.com/neuvector/neuvector/security/advisories/GHSA-8ff6-pc43-jwv3[CVE-2025-53884] 
| NeuVector utilise un sel cryptographiquement sécurisé avec l'algorithme PBKDF2 au lieu d'un simple hachage pour protéger les mots de passe des utilisateurs. Pour les mises à jour de versions antérieures, NeuVector recalcule et stocke le nouveau hachage du mot de passe seulement après la prochaine connexion réussie de chaque utilisateur.
| 25 août 2025 
| https://github.com/neuvector/neuvector/releases/tag/v5.4.6[NeuVector v5.4.6]

Les utilisateurs peuvent configurer un ConfigMap Kubernetes pour définir des modèles de regex personnalisés pour les commandes de processus supplémentaires à expurger.
| 25 août 2025

== Exposition d'informations sensibles dans les journaux de conteneurs de NeuVector Manager

*CVE ID :* CVE-2025-46808*Score CVSS :* 6.8 - https://nvd.nist.gov/vuln-metrics/cvss/v3-calculator?vector=AV:N/AC:L/PR:L/UI:R/S:C/C:H/I:N/A:N&version=3.1[AV:N/AC:L/PR:L/UI:R/S:C/C:H/I:N/A:N]*CWE :* https://cwe.mitre.org/data/definitions/532[: Insertion d'informations sensibles dans un fichier journal] CWE-532

=== Versions concernées

* Toutes les versions antérieures à `+5.0.0+`
* Versions à partir de `+5.0.0+` jusqu'à et y compris `+5.4.4+`

*Version corrigée : `+5.4.5+`*

=== Impact

Une vulnérabilité a été identifiée dans les versions de NeuVector jusqu'à `+5.4.4+`, où des informations sensibles sont divulguées dans le journal du conteneur de gestion. Les champs suivants peuvent être exposés :

[cols="1,2,2,2,2", options="header"]
|===
|Champ d'application |Description du champ |Où il apparaît |Reproduction |Environnement

|`+X-R-Sess+`
|Jeton de session de Rancher pour l'authentification unique
|En-tête de la demande
|Se connecter via l'interface Rancher et accéder à NeuVector SSO
|Rancher avec NeuVector SSO

|`+personal_access_token+`
|GitHub / Azure DevOps token
|Corps de la demande
|Soumettre la configuration du dépôt à distance sous *Configuration > Paramètres*
|NeuVector

|`+token1.token+`
|Jeton de session de l'utilisateur de NeuVector
|Organe de réponse
|Demande GET à `+https://<neuvector-ui-url>/user?name=<username>+`
|NeuVector

|`+rekor_public_key+`, `+root_cert+`, `+sct_public_key+`
|Clé publique de Rekor, certificat racine, clé publique de SCT dans la racine de confiance privée
|Corps de la demande
|Créer/mettre à jour la racine de confiance privée dans la page Sigstore
|NeuVector

|`+public_key+`
|Clé publique du vérificateur
|Corps de la demande
|Créer/mettre à jour le vérificateur dans la page Sigstore
|NeuVector
|===

[NOTE]
====
Les installations de NeuVector qui n'utilisent pas Rancher SSO ou Remote Repository Configuration ne sont *pas concernées* par ce problème.
====

Dans la version corrigée, la valeur de `+X-R-Sess+` est partiellement masquée pour des raisons de sécurité. D'autres champs tels que `+personal_access_token+`, `+token+`, `+rekor_public_key+`, `+root_cert+`, `+sct_public_key+`, et `+public_key+` ne sont plus enregistrés, car l'enregistrement du corps de la requête a été supprimé.

[NOTE]
====
* La gravité de cette vulnérabilité dépend de votre stratégie de journalisation :
  ** *Journalisation locale (par défaut) :* Limite l'exposition à l'impact.
  ** *Journalisation externe :* Augmente le risque, en fonction du niveau de sécurité de votre collecteur de journaux externe.
* La gravité de l'impact peut également dépendre des autorisations des informations d'identification exposées.

Voir aussi : https://attack.mitre.org/techniques/T1552/[MITRE Technique T1552 : Unsecured Credentials (informations d'identification non sécurisées]).
====

=== Les écussons

Les versions corrigées comprennent `+5.4.5+` et les versions supérieures. Les utilisateurs doivent changer tout jeton GitHub utilisé dans la configuration du dépôt à distance après la mise à jour.

=== Solutions de rechange

Aucune solution de contournement n'est actuellement disponible. Passez à la version corrigée dès que possible.
