== Testen der Aktivierung von Chrome-Browsern zur Annahme des \{product-name} selbstsignierten Zertifikats auf MacOS

Unter bestimmten Umständen wird der Chrome-Browser einfach ablehnen, ein \{product-name} selbstsigniertes Zertifikat zu akzeptieren.  Ein möglicher Grund könnte sein, dass das Zertifikat noch nicht in den macOS-Anmeldezertifikatspeicher importiert wurde.  Eine weitere Möglichkeit ist, dass ein vorhandenes Zertifikat noch nicht als vertrauenswürdig konfiguriert wurde.  Dies wird dem Benutzer verbieten, es zu akzeptieren und zur \{product-name} Anmeldeseite mit Chrome fortzufahren, wie im untenstehenden Beispiel dargestellt:

image:chrome1.png[chrome]

Korrekturmaßnahmen können ergriffen werden, um Chrome zu aktivieren, um das selbstsignierte Zertifikat im macOS-Schlüsselbund zu akzeptieren.  Dies kann erreicht werden, indem das Zertifikat des \{product-name} so konfiguriert wird, dass es mit der Mac-Schlüsselbund-Zugriffsanwendung als vertrauenswürdig gilt.  Dies setzt voraus, dass das Zertifikat tatsächlich im macOS-Schlüsselbund-Zugriffspeicher vorhanden ist.  Wir werden zunächst betrachten, wie man ein Zertifikat von einem vorhandenen \{product-name} Knoten von einem System exportiert und später in ein anderes System importiert.   Es gibt 2 Möglichkeiten, das Zertifikat zu exportieren, die wir im Folgenden durchgehen werden.  Sie sind, den Chrome-Browser zu verwenden und die macOS-Schlüsselbund-Zugriffsanwendung zu verwenden.

=== Testen des Zertifikatexports mit macOS Schlüsselbund-Zugriff

Starten Sie die macOS Schlüsselbund-Zugriffsanwendung, indem Sie gleichzeitig die Tasten ⌘ + F auf einer Mac-Tastatur eingeben, suchen Sie nach Schlüsselbund-Zugriff und doppelklicken Sie auf die Anwendung.
Dies startet die Anwendung Schlüsselbundzugriff.  Wählen Sie das \{product-name} Zertifikat aus und wählen Sie Export "`{product-name}`" aus dem Dropdown-Menü.  Dies ermöglicht den Export des Zertifikats an einen gewünschten Ort.

image:chrome4.png[chrome]

=== Test Starten der macOS Schlüsselbundzugriffs-Anwendung

Es gibt zwei Möglichkeiten, die macOS Schlüsselbundzugriffs-Anwendung zu starten.  Dies kann durch Suchen im macOS erfolgen, wie in den oben beschriebenen Schritten, oder durch die Verwendung des Chrome-Browsers.

=== Test Starten des Schlüsselbundzugriffs aus dem Chrome-Browser

Einstellungen > Erweitert > Zertifikate verwalten

image:chrome5.png[chrome]

Ein Klick auf die Zertifikate verwalten, wie oben gezeigt, startet die Schlüsselbundzugriffs-Anwendung des Macs, wie unten gezeigt.

image:chrome6.png[chrome]

=== Test Import des Zertifikats in den macOS Schlüsselbundzugriffs-Speicher

Starten Sie die macOS Schlüsselbundzugriffs-Anwendung, indem Sie gleichzeitig die Tasten ⌘ + F auf einer Mac-Tastatur eingeben, suchen Sie nach Schlüsselbundzugriff und klicken Sie auf die Anwendung.  Importieren Sie dann das Zertifikat, indem Sie ein \{product-name} Zertifikat (Beispiel: nvcertificate.cer) in den rechten Bereich des Zertifikats der Schlüsselbundzugriffs-Anwendung ziehen und ablegen.  Jetzt, da das Zertifikat importiert wurde, ist es noch nicht vertrauenswürdig.
Das folgende Beispiel zeigt die Standardvertrauenseinstellungen, nachdem das Zertifikat eines \{product-name} in die Schlüsselbundzugriffs-Anwendung importiert wurde.

image:chrome7.png[chrome]

Beachten Sie, dass keine der Parameter konfiguriert wurden und es standardmäßig nicht vertrauenswürdig ist.  Es muss konfiguriert werden, um vertrauenswürdig zu sein, damit der Benutzer auf die Anmeldeseite mit dem Chrome-Browser zugreifen kann.

=== Test Aktivierung des Vertrauens für das \{product-name} Zertifikat

Die folgenden Schritte erläutern, wie das Vertrauen für das \{product-name} Zertifikat im Schlüsselbundzugriffs-Speicher aktiviert werden kann.

image:chrome8.png[chrome]

Ein Doppelklick auf das \{product-name} Zertifikat aus dem obigen Screenshot öffnet ein weiteres Popup-Dialogfeld, das es ermöglicht, die Vertrauensberechtigungen für verschiedene Parameter zu konfigurieren.  Konfigurieren Sie Secure Sockets Layer (SSL) auf "`Immer vertrauen`".

image:chrome9.png[chrome]

Schließen Sie das Dialogfeld und klicken Sie auf '\`Einstellungen aktualisieren`", nachdem Sie das Passwort des Mac-Anmeldbenutzers aus dem folgenden Popup-Dialogfeld eingegeben haben.

image:chrome10.png[chrome]

Dies finalisiert die Vertrauenseinstellungen.  Als Nächstes löschen Sie den gesamten Cache des Chrome-Browsers, schließen alle Chrome-Browser und starten den Chrome-Browser neu.  Besuchen Sie als Nächstes die Anmeldeseite von \{product-name}, und nach dem Klicken auf den "`Erweitert`" Link wird der Link "Weiter zu ... (unsicher)" angezeigt.  Ein Klick auf "Weiter zu ... (unsicher)" ermöglicht es dem Benutzer, zur Anmeldeseite für Benutzername und Passwort zu gelangen.

image:chrome11.png[chrome]

Unten sehen Sie einen Screenshot, der es dem Benutzer ermöglicht, zur Anmeldeseite von \{product-name} zu gelangen, nachdem der Link "Weiter zu ... (Unsicher)" angeklickt wurde.
