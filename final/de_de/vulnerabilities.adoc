== Test_DreiVerwaltung von Schwachstellen mit \{product-name}

\{product-name} ermöglicht automatisiertes Scannen und Management von Schwachstellen über die gesamte Pipeline. Best Practices für das Management von Schwachstellen in \{product-name} umfassen:

* Scannen während der xref:build-image-scanning.adoc[Build-Phase], wobei der Build fehlschlägt, wenn kritische Schwachstellen 'mit verfügbarer Lösung' vorhanden sind. Dies zwingt Entwickler, behebbare Schwachstellen zu beheben, bevor sie in Registries gespeichert werden.
* Scannen Sie Staging- und Produktionsregistries kontinuierlich, um nach neu entdeckten Schwachstellen zu suchen. Schwachstellen mit verfügbaren Lösungen können sofort behoben werden müssen, oder es kann eine Karenzzeit gewährt werden, um Zeit zur Behebung zu bieten.
* Konfigurieren Sie xref:admission.adoc[Zulassungssteuerung]-Regeln, um Bereitstellungen in die Produktion basierend auf Kriterien wie kritisch/hoch, verfügbare Lösung und gemeldetes Datum zu blockieren.
* Scannen Sie kontinuierlich die Produktionsknoten/Hosts, Container und die Orchestrierungsplattform auf neu entdeckte Schwachstellen. Implementieren Sie Reaktionen basierend auf Kritikalität/Schwere, die Webhook-Benachrichtigungen (die Sicherheit und Entwickler kontaktieren), Quarantäne-Container oder den Beginn einer Karenzzeit zur Behebung umfassen können.
* Stellen Sie sicher, dass laufende Container im Überwachungs- oder Schutzmodus mit geeigneten Whitelist-Regeln sind, um Schwachstellen 'virtuell zu patchen', um Exploits in der Produktion zu verhindern.
* Scannen Sie distroless und PhotonOS-basierte Images.

Das Dashboard in \{product-name} präsentiert eine zusammenfassende Risikobewertung, die Schwachstellen umfasst und zur Risikominderung von Schwachstellen verwendet werden kann. Siehe xref:improve-score.adoc[wie man die Risikobewertung verbessert] für weitere Details.

Das andere Hauptwerkzeug zur Überprüfung, Filterung und Berichterstattung über Schwachstellen befindet sich im Menü Sicherheitsrisiken.

=== Test_DreiSicherheitsrisiken Menü - Schwachstellen

Dieses Menü kombiniert die Ergebnisse aus den Registrierungs- (Image-), Knoten- und Container-Schwachstellenscans sowie Compliance-Prüfungen, die im Menü Assets gefunden wurden, um ein End-to-End-Management und Reporting von Schwachstellen zu ermöglichen.

Das Menü für Schwachstellen bietet ein leistungsstarkes Erkundungstool, um:

* Es einfach zu machen, Berichte anzuzeigen oder herunterzuladen, indem Sie eine Suchzeichenfolge eingeben oder den erweiterten Filter neben dem Feld verwenden. Der erweiterte Filter ermöglicht es Benutzern, Schwachstellen nach verfügbaren (oder nicht verfügbaren) Fixes, Dringlichkeit, Arbeitslasten, Dienst, Container, Knoten oder Namespace-Namen zu filtern.
* Die Auswirkungen von Schwachstellen und Compliance-Prüfungen zu verstehen, indem Sie auf die Impact-Zeile klicken und die Behebung sowie betroffene Bilder, Knoten oder Container überprüfen.
* Den Schutzstatus (Ausnutzungsrisiko) einer Schwachstelle oder Compliance-Problematik anzeigen, um zu sehen, ob für betroffene Knoten oder Container \{product-name} Run-Time-Sicherheitsmaßnahmen (Regeln) aktiviert sind.
* 'Akzeptieren' Sie eine Schwachstelle/CVE, nachdem sie überprüft wurde, um sie aus Ansichten auszublenden und in Berichten zu unterdrücken.

image:vulnerabilities_4_4.png[SecurityRisks]

Verwenden Sie das Filterfeld, um eine Zeichenfolgenübereinstimmung einzugeben, oder verwenden Sie den erweiterten Filter daneben, um spezifischere Kriterien auszuwählen, wie unten gezeigt. Heruntergeladene PDF- und CSV-Berichte zeigen nur die gefilterten Ergebnisse an.

image:advanced_filter_4.png[AdvancedFilter]

Die Auswahl einer aufgeführten CVE bietet zusätzliche Details zur CVE, zur Behebung und zu den betroffenen Bildern, Knoten oder Containern. Das Symbol für den Schutzstatus (Kreis) zeigt verschiedene Farben an, um einen groben Prozentsatz der betroffenen Elemente anzuzeigen, die zur Laufzeit von \{product-name} ungeschützt sind, durch \{product-name}-Regeln (im Monitor- oder Schutzmodus) geschützt sind oder zur Laufzeit nicht betroffen sind (z. B. ein Bild, das mit dieser Schwachstelle gescannt wurde, hat keine laufenden Container). Das Farbschema der Schutzstatus-Spalte ist:

* Black = Test_Threeunaffected
* Grün = Test_Threegeschützt durch \{product-name} im Monitor- oder Schutzmodus
* Rot = Test_Threeungeschützt durch \{product-name}, noch im Entdeckungsmodus

Das Farbschema des Impact-Analysefensters (zeigt betroffene Bilder, Knoten, Container) ist:

* Black = Test_Threeunaffected. Es gibt keine Container, die dieses Bild in der Produktion verwenden
* Lila = Test_Threelaufend im Monitor-Modus in der Produktion
* Dunkelgrün = Test_Threelaufend im Schutzmodus in der Produktion
* Hellblau = Test_Threelaufend im Entdeckungsmodus in der Produktion (ungeschützt)

Die Impact-Farben sollen den Laufzeitschutzfarben für die Discover-, Monitor- und Protect-Modi an anderen Stellen in der \{product-name}-Konsole entsprechen.

=== Test_DreiAkzeptierende Schwachstellen

Sie können eine Schwachstelle (CVE) 'Akzeptieren', um sie von Berichten, Ansichten, Risikobewertungen usw. auszuschließen. Eine Schwachstelle kann ausgewählt und die Schaltfläche Akzeptieren von mehreren Bildschirmen aus angeklickt werden, wie z.B. Sicherheitsrisiken -> Schwachstellen, Assets -> Container usw. Sobald sie akzeptiert wurde, wird sie zur Liste des Sicherheitsrisiken -> Schwachstellenprofils hinzugefügt. Sie kann hier angezeigt, exportiert und bearbeitet werden. Beachten Sie, dass diese Akzeptieren-Funktion auf aufgeführte Bilder und/oder Namespaces beschränkt werden kann. Neue Einträge können auch manuell von diesem Bildschirm aus zu dieser Liste hinzugefügt werden.

Um eine Schwachstelle global zu akzeptieren, gehen Sie zur Seite Sicherheitsrisiken -> Schwachstellen und wählen Sie die Schwachstelle aus, und klicken Sie dann auf Akzeptieren. Dies wird ein Schwachstellenprofil für diese CVE global erstellen.

image:accept_global.png[global]

Um eine Schwachstelle zu akzeptieren, die in einem Bildscan gefunden wurde, öffnen Sie die Bildscan-Ergebnisse in Assets -> Registries, ziehen Sie das Dropdown-Menü Ansicht nach unten und wählen Sie Akzeptieren. Beachten Sie, dass Sie auch wählen können, akzeptierte Schwachstellen für dieses Bild anzuzeigen oder auszublenden. HINWEIS: Diese Aktion wird einen Eintrag im Schwachstellenprofil für diese CVE nur in diesem BILD erstellen.

image:accept_image_cve.png[image]

Um eine Schwachstelle zu akzeptieren, die in einem Container in Assets -> Container gefunden wurde, wählen Sie die Schwachstelle aus, ziehen Sie das Dropdown-Menü Ansicht nach unten und wählen Sie Akzeptieren. Beachten Sie, dass Sie auch wählen können, akzeptierte Schwachstellen für diesen Container anzuzeigen oder auszublenden. HINWEIS: Diese Aktion wird ein Schwachstellenprofil für diese CVE nur in diesem NAMESPACE erstellen.

image:accept_containers.png[container]

Diese Aktion kann auch in Assets -> Nodes durchgeführt werden, was ein Schwachstellenprofil für die CVE für alle Container, Bilder und Namespaces erstellt.

[IMPORTANT]
====
Weltweit akzeptierte Schwachstellen sind in der Ansicht der Sicherheitsrisiken -> Schwachstellen und in exportierten Berichten von dieser Seite ausgeschlossen. Akzeptierte Schwachstellen, die auf bestimmte Images oder Namespaces beschränkt sind, werden weiterhin in der Ansicht angezeigt, aber in Berichten ausgeschlossen, in denen der erweiterte Filter die Ansicht auf diese Images oder Namespaces beschränkt.
====

=== Test_DreiVerwaltung von Schwachstellenprofilen

Jede akzeptierte Schwachstelle/CVE erstellt einen Eintrag in der Liste der Sicherheitsrisiken -> Schwachstellenprofile. Diese Einträge können bearbeitet werden, um Attribute wie Bildnamen und Namespaces hinzuzufügen oder zu entfernen.

image:vuln_profile.png[profil]

Neue akzeptierte Schwachstellen können auch hier hinzugefügt werden, indem der CVE-Name eingegeben wird, der akzeptiert werden soll.
