== LDAP

Configurez les champs requis pour vous connecter à votre serveur LDAP.

image:ldap-ad.png[LDAPAD]

. Port. Le port par défaut est 389 pour SSL désactivé et 636 pour SSL activé.
. Nom d'utilisateur (facultatif). Nous utilisons ce nom d'utilisateur administrateur pour lier le serveur ldap pour chaque requête.
. Base dn. Cela devrait être un nœud racine dans le serveur ldap pour rechercher l'utilisateur et le groupe ldap.
. Rôle par défaut. C'est le rôle qu'un utilisateur prendra si le mappage de groupe de rôle (ci-dessous) échoue. Si l'attribut de groupe de l'utilisateur est trouvé et peut être mappé à un rôle, alors le rôle par défaut ne sera pas utilisé. Si aucun attribut de groupe correspondant n'est trouvé, le rôle par défaut sera pris. Si le rôle par défaut est Aucun dans ce cas, la connexion de l'utilisateur échouera. Le bouton '`test connection`' vérifiera si un nom d'utilisateur/mot de passe peut être authentifié par le serveur LDAP configuré.
. Mappage des rôles Admin et Reader. Cela définit comment mapper l'appartenance au groupe LDAP d'un utilisateur au rôle utilisateur dans {product-name}. Ajoutez la liste des groupes LDAP aux rôles correspondants. Lors de la recherche de l'appartenance au groupe d'un utilisateur dans le schéma LDAP, nous supposons que l'attribut membre du groupe est nommé "`memberUid`".

=== Mappage des groupes aux rôles et espaces de noms
