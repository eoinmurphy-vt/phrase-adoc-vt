== Améliorer le score de risque de sécurité

Le score de risque de sécurité dans le tableau de bord fournit une note entre 0 et 100.

* 0-20 Bon
* 21-50 Juste
* 51-100 Faible

La note est la somme des paramètres suivants, chacun étant représenté par une valeur maximale, avec un maximum de 100 :

* {product-name} Mode de protection - 30
* Risque d'entrée et de sortie - 42
* Conteneurs privilégiés - 4
* Conteneurs de racines - 4
* Contrôles d'admission - 4
* Vulnérabilités - 16 (conteneurs - 8, hôte - 6, plateforme orchestrator - 2)

Par défaut, {product-name} inclut tous les conteneurs, y compris les conteneurs système, dans le score de risque. Cette option peut être personnalisée pour chaque groupe de conteneurs appris afin de désactiver certains conteneurs dans le calcul de la note de risque, comme indiqué ci-dessous.

=== Comment améliorer le score

==== {product-name} Mode de protection

. Modifier le mode de protection du nouveau service dans Paramètres -> Configuration pour surveiller ou protéger
. Modifier tous les groupes '`learned`' dans la politique -> Groupes à surveiller ou à protéger

Ou

. Cliquez sur l'icône de l'outil pour suivre l'assistant afin d'effectuer les étapes ci-dessus.

==== Risque lié aux entrées et sorties

. Cliquez sur l'icône de l'outil pour suivre l'assistant afin d'examiner les entrées et les sorties.
. Examinez toutes les connexions d'entrée et de sortie pour vous assurer qu'elles sont autorisées.
. Tous les services qui sont encore en mode découverte passent en mode surveillance ou protection.
. Examinez et effacez toutes les menaces, les violations et l'historique de la session en cliquant sur l'icône Supprimer/Corbeille pour chacune d'entre elles.

==== Conteneurs privilégiés et/ou racine

. Retirer les conteneurs privilégiés
. Supprimer les conteneurs de racines


[NOTE]
====
Il se peut que cela ne soit pas possible en raison des conteneurs requis, mais chacun d'entre eux ne compte que pour 4 points.
====

==== Contrôles d'admission

. Assurez-vous que, dans un environnement Kubernetes/OpenShift, le contrôle d'admission est activé et qu'il y a au moins une règle active dans la politique -> Contrôle d'admission.

==== Vulnérabilités

. Assurez-vous que tous les conteneurs hors système sont en mode Surveillance ou Protection, dans Policy -> Groups.
. Supprimer/remédier les vulnérabilités de l'hôte
. Supprimer/remédier les vulnérabilités de la plateforme d'orchestration (par exemple Kubernetes, OpenShift).

=== Comment personnaliser les groupes de conteneurs inclus dans le score ?

Pour activer ou désactiver les groupes de conteneurs inclus dans le Security Risk Score, allez dans le menu Policy -> Groups et sélectionnez le groupe à modifier. La colonne de résumé à droite comporte une icône "Scorable" qui indique les groupes utilisés pour la notation.

image:risk_scorable.png[écoresponsable]

Cochez ou décochez la case Scorable en haut à droite pour le groupe sélectionné.

[NOTE]
====
Seuls les "groupes appris" (par exemple ceux qui commencent par "nv.") peuvent être modifiés, pas les groupes réservés ni les groupes personnalisés.
====
